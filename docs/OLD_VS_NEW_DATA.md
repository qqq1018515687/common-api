## 元数据对比

### 旧数据元数据

```json
{
  "Metadata": {}
}
```

**说明**：旧数据没有元数据，无法判断文件类型、创建时间、过期时间。

### 新数据元数据

```json
{
  "Metadata": {
    "category": "upload",
    "created_at": "1771218000",
    "expires_in": "604800",
    "original_filename": "avatar_123.png",
    "is_permanent": "False"
  }
}
```

**说明**：新数据有完整的元数据，可以自动判断和清理。

## 清理行为对比

### 旧数据的清理行为

```python
# 清理工具处理旧数据的逻辑
for file_key in old_files:
    metadata = storage_mgr.get_file_metadata(file_key)
    
    # 旧数据无元数据
    if not metadata:
        logger.info(f"跳过无元数据文件: {file_key}")
        continue  # 跳过，不删除
```

**结果**：❌ 旧数据不会被自动清理

### 新数据的清理行为

```python
# 清理工具处理新数据的逻辑
for file_key in new_files:
    metadata = storage_mgr.get_file_metadata(file_key)
    
    # 新数据有元数据
    if metadata:
        is_expired = storage_mgr.is_expired(file_key)
        
        if is_expired:
            # 删除过期文件
            storage.delete_file(file_key=file_key)
```

**结果**：✅ 新数据会自动清理过期文件

## 决策指南

### 是否需要迁移旧数据？

#### ✅ 需要迁移的情况

1. 需要自动清理旧数据
2. 需要统一管理所有文件
3. 需要了解旧数据的创建时间
4. 需要按分类批量操作旧数据

#### ❌ 不需要迁移的情况

1. 旧数据量很少，手动管理即可
2. 旧数据都是重要文件，不希望自动清理
3. 担心迁移过程中的数据丢失风险
4. 旧数据即将过期，等待自然失效即可

### 推荐策略

**保守策略**（推荐）：
```
1. 新文件自动使用新方案
2. 旧文件保持不变，继续正常工作
3. 定期手动清理不需要的旧文件
4. 如需要，可选择性地迁移部分旧文件
```

**激进策略**：
```
1. 先试运行迁移，检查将要迁移的文件
2. 确认无误后，实际迁移所有旧数据
3. 迁移后使用清理工具自动管理
```

## 总结

### 关键要点

1. **旧数据不会自动分类**
   - 旧数据：`avatar_123.png`（无分类前缀）
   - 新数据：`uploads/avatar_123.png`（有分类前缀）

2. **旧数据无法自动删除**
   - 旧数据没有元数据
   - 清理工具会跳过旧数据

3. **新旧数据互不影响**
   - 旧数据继续正常工作
   - 新数据自动分类管理
   - 可以选择性地迁移旧数据

4. **迁移是可选的**
   - 不迁移也能正常使用
   - 迁移可以获得更好的管理能力
   - 建议先试运行，确认后再执行

### 建议行动

1. **立即执行**：新文件自动使用新方案
2. **定期检查**：使用 `--analyze` 查看旧数据情况
3. **按需迁移**：根据实际需求选择是否迁移旧数据
4. **谨慎操作**：迁移前先试运行，确认无误后再执行
