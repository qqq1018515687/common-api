# 安全清理旧数据指南

## 目标

只保留头像，删除其他旧数据（上传、临时文件等）

## 文件识别规则

### 头像文件（保留）

符合以下条件的文件会被识别为头像并保留：

1. **文件名以 `avatar_` 开头**
   ```
   avatar_abc12345.png ✓
   avatar_def67890.jpg ✓
   ```

2. **文件名包含 `avatar` 关键字**
   ```
   user_123_avatar.png ✓
   avatar_user_456.jpg ✓
   ```

### 其他文件（删除）

不符合头像条件的旧文件会被删除：

```
upload_ghi01234.png ✗
temp_jkl56789.bin ✗
image_mno90123.jpg ✗
file_pqr34567.pdf ✗
```

### 已有分类前缀的文件（不受影响）

新上传的文件已有分类前缀，不受影响：

```
avatars/avatar_xyz123.png  ✓  不受影响（新数据）
uploads/upload_456.png    ✓  不受影响（新数据）
temp/temp_789.bin         ✓  不受影响（新数据）
```

- [ ] 已执行 `--cleanup`（试运行），确认将要删除的文件列表

- [ ] 确认其他文件确实不需要（会被永久删除）
- [ ] 如有重要文件，先手动备份

## 常见问题

### Q1: 如何判断一个文件是否会被保留？

**A**: 检查文件名：
- 包含 `avatar` 字样 → 保留
- 不包含 `avatar` 字样 → 删除

```bash
# 查看具体列表
python scripts/clean_old_data.py --analyze --verbose
```

### Q2: 头像文件会被误删吗？

**A**: 不会。只有以下文件会被删除：
- 旧数据（无分类前缀）
- 文件名不包含 `avatar` 字样

头像文件只要文件名包含 `avatar`，就会被保留。

### Q3: 新上传的文件会被删除吗？

**A**: 不会。新上传的文件有分类前缀（`avatars/`、`uploads/`、`temp/`），不会被清理工具处理。

### Q4: 如果我想保留某些非头像的旧文件怎么办？

**A**: 有两种方法：

**方法 1**: 手动重命名文件（添加 `avatar` 字样）
```bash
# 例如：将 upload_important.png 重命名为 avatar_upload_important.png
```

**方法 2**: 手动迁移到新方案
```bash
python scripts/migrate_old_data.py --analyze
python scripts/migrate_old_data.py --migrate
```

### Q5: 清理后可以恢复吗？

**A**: 不可以。对象存储通常不支持恢复已删除的文件。

**建议**：
- 清理前先试运行，确认文件列表
- 如有重要文件，先手动备份
- 在低峰期执行清理

### Q6: 清理失败怎么办？

**A**: 查看清理日志，失败的文件会被列出：

```
清理结果:
  失败: 2

删除失败的文件:
  - upload_failed_1.png
  - upload_failed_2.jpg
```

可能的原因：
- 网络问题
- 权限问题
- 文件已被删除

解决方法：
- 检查网络连接
- 检查存储权限
- 重新执行清理

### Q7: 能否只清理特定类型的文件？

**A**: 当前版本只支持"保留头像，删除其他"的模式。

如果需要更精细的控制，可以：
1. 使用 `--analyze --verbose` 查看所有文件
2. 手动选择要删除的文件
3. 通过对象存储控制台手动删除

## 总结

### 核心要点

1. **安全第一**：先试运行，确认无误后再执行
2. **只删除非头像**：头像文件自动保留
3. **新数据不受影响**：已有分类前缀的文件不受影响
4. **无法恢复**：删除后无法恢复，请谨慎操作

### 推荐流程

```
1. python scripts/clean_old_data.py --analyze
   ↓
2. python scripts/clean_old_data.py --analyze --verbose（可选）
   ↓
3. python scripts/clean_old_data.py --cleanup（试运行）
   ↓
4. 确认无误后，输入 yes
   ↓
5. python scripts/clean_old_data.py --cleanup --force（实际执行）
```

### 预期效果

- ✅ 释放存储空间
- ✅ 清理无用文件
- ✅ 保留重要头像
- ✅ 不影响新数据

准备好后，按照步骤操作即可！
